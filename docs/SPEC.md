# アプリケーション仕様書 (SPEC.md)

## 1. ターゲットユーザー
- 大学の講義内容を効率的に復習したい学生。
- 会議の議事録作成を短縮したい社会人。

## 2. ユーザーフロー
1. ユーザーがAPIキーを入力（初回のみ、localStorageに保存）。
2. 音声ファイルをファイル選択ボタンから投入。
3. 解析ボタン押下により、Base64形式でGemini APIへ送信。
4. 解析結果（文字起こし・要約・要点）を画面に表示。

## 3. 機能要件
- **音声入力**: ファイルアップロード形式（.m4a 対応）。
- **AI連携**: `gemini-1.5-flash` モデルを使用。
- **UI**: モバイルファーストのレスポンシブデザイン。
- **セキュリティ**: APIキーをコードに含めず、ユーザー入力形式を採用。

## 開発の記録と詰まった点
1. **APIバージョンとモデルの不一致 (404 Error)**
   - **状況**: 初期実装時に `v1beta` と `v1` の使い分け、およびモデル名の指定方法が原因で404エラーが多発した。
   - **解決**: AIとの対話を通じ、最新の公式ドキュメントに基づいたエンドポイント（v1）とモデル名（gemini-1.5-flash）の正しい組み合わせを特定し、コードを修正して解決した。

2. **ファイル形式の制限への対応**
   - **状況**: Windowsの標準録音形式（.m4a）をアップロードした際、AI側で正しく認識されないケースがあった。
   - **解決**: MIMEタイプを `audio/mp4` として明示的に扱うようにプログラムを調整し、汎用性を高めた。

3. **外部AIのデバッグ活用**
   - **状況**: 自分一人では原因特定が難しいエラーに直面した。
   - **解決**: 外部AIに対し、具体的なエラーログと最新のAPI仕様を前提としたプロンプトを投げることで、短時間で動作するコードへ改善することができた。
